# data/ — inputs, provenance, and where the rest lives

This folder holds the **small, tracked** data the SEPIA site/analyses read directly.
Large and cluster-only data are **not** here (they're gitignored or live on Great Lakes /
in the shared lab genomes dir); this file documents *where* each of those lives so nothing
is a mystery. Rule of thumb: if a file is big or binary genomic (BAM/BCF/VCF/plink), it is
**not** in git — see the tables below for its real location.

Provenance for the archaic genomes specifically (per-genome source paper, DOI, coverage,
download URLs) is in [`../plan/papers/REFERENCES.md`](../plan/papers/REFERENCES.md); how the
pipeline transforms them is in [`../plan/pipeline.html`](../plan/pipeline.html).

| File | What it is | Source / how produced | Status |
|---|---|---|---|

| `snps.txt` | The pigmentation SNP list (222-SNP panel identifiers) | derived from `skin_pigmentation.tsv` | **keep** (input) |

| `sgdp.wg.pca.eigenvec` / `.eigenval` | Old **modern** PCA (only 15 SGDP individuals) | Lily's `sgdp_merge_pca.slurm` (incomplete merge) | **legacy / buggy** — superseded by the 166-sample rebuild (`output/pca_wg/`) |
| `pigmentation.pca.*`, `pigmentation_snps_pca.*` | Old pigmentation-panel PCA (rank-1 / degenerate) | Lily's pigmentation PCA | **legacy / buggy** — kept only as before/after evidence |
| `ancient.projected.*.sscore*`, `*.projected.sscore.tmp` | Old archaic projections onto the buggy PCAs | Lily's projection step | **legacy / buggy** |
| `sgdp_metadata.tsv` | SGDP sample → population/region labels (for colouring PCAs) | public SGDP metadata (Mallick et al. 2016, Reich Lab, Harvard) — full provenance in `sgdp_metadata.SOURCE.md` | **keep** (input) |

The `legacy / buggy` files are retained deliberately as the "before" side of the
before/after comparison (see [`../plan/verify/BUG_EVIDENCE.md`](../plan/verify/BUG_EVIDENCE.md));
SEPIA's replacements are regenerated by the pipeline and land in `../output/`.
