2026-07-23 — Re-scoped to a genome-wide sanity PCA first; found the modern reference is a 15-of-166 partial merge

  • DECIDED (Tina) — before any pigmentation-specific analysis, run a genome-wide PCA of ancient + modern samples first — "just for sanity, to see where everyone clusters" — then the pigmentation-loci PCA later. Do basic QC ourselves. Do not optimize a narrow panel-only extraction.
  • DONE — Cancelled Stage 0 (job 54578430). It was downloading whole-chromosome all-sites snpAD VCFs (2–4 GB each × 4 genomes × 23 chr ≈ ~275 GB) just to read 222 panel positions — the wrong scope for a genome-wide PCA, and it would overflow the 110 GB working dir. Killed at 12 GB.
  • FOUND (validation, kept for the later panel step) — a remote region query works: bcftools view -r 15:48426484 <EVA-URL> returned Denisova 3 at SLC24A5 with DP=21 in 4.8 s (KB fetched, no bulk download). Confirms the published high-coverage Denisova 3 calls are genuine (the A1 fix) and can be pulled per-site when we get to the panel PCA.
  • FOUND (the real blocker) — the genome-wide data already exists on the cluster and is build-consistent (hg19): pca/modern/sgdp.wg (modern, 9.23M SNPs) + ancient_wholegenome/ancient_sgdp_wg.vcf.gz (15 archaics at the SGDP SNP set, 33.5M sites; tracer 15:48426484 ✓, bare-contig GRCh37 ✓). But pca/modern/sgdp.wg holds only 15 individuals, while 166 per-sample SGDP BCFs sit in sgdp/subsets/ with a merge_continue.slurm present — the modern PCA reference is an incomplete/partial merge. Fifteen individuals cannot define global continental structure — almost certainly why earlier PCAs looked degenerate. Completing the 166-sample merge is the prerequisite for any meaningful ancient-vs-modern PCA.
  • FOUND (ancient calls are pseudo-haploid) — ancient_wg.slurm calls the archaics bcftools call -m --ploidy 1 (haploid) at -q30 -Q30 from the sorted BAMs. So individual heterozygosity cannot be computed from ancient_sgdp_wg (no diploid genotypes) → het-vs-depth (Q7) must come from ANGSD on the BAMs. For PCA, the pseudo-haploid ancients should be projected onto the diploid modern PCA (plink2 --pca allele-wts → --score … no-mean-imputation), which also delivers the Q5 no-mean-imputation fix.
  • FOUND (sample-name hygiene, Q7) — ancient_sgdp_wg.vcf.gz sample IDs are a mix of full BAM paths and SM-tag names (Vi33.19, L4741, SP4903) — a labeling bug to fix by reheader before merging.
  • FOUND (degenerate-PCA mechanism, pigmentation branch) — pca_ancient.slurm uses a different reference (reference/hg19.fa, chr-named) than the WG branch (hs37d5.fa, bare) = the A2 build/naming split, and runs bcftools call -mv (variants-only) on ancients only at panel sites → monomorphic panel SNPs dropped, no modern reference merged in → rank-deficient PCA. Confirms A2 + the projection collapse are isolated to the pigmentation branch and do not contaminate the WG data.
  • PREPARED (held) — wrote wg_modern_merge.slurm to complete the modern SGDP merge from all 166 subset BCFs → LD-prune (--indep-pairwise 200 25 0.4) → plink2 --pca 10 allele-wts + --freq (projection basis). Not yet submitted — held pending the project rename + cluster reorganization below (its output paths move to sepia/).
  • NEXT — reheader + intersect the ancient calls to the pruned modern SNP set, project them onto the modern PCA with no-mean-imputation (pseudo-haploidize the modern side for a like-for-like comparison), per-sample call-rate/coverage QC (drops the near-empty low-cov archaics), then the "where everyone clusters" figure. Then ANGSD het-vs-depth; then the pigmentation-loci PCA done correctly.
